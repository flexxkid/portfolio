Featured Project

GardaWorld
HR Management System

Digitizing HR operations for Kenya's leading security firm — from paper filing cabinets to a secure, centralized digital platform.

Client GardaWorld Security Kenya
Stack Laravel · React · MySQL · AWS S3
Type Final Year Project
Status In Progress — 2025
01 — The Problem

Thousands of records. Zero digital infrastructure.

GardaWorld Security Kenya manages a large, distributed workforce of security personnel deployed across multiple branches. Every employee record — qualifications, licences, deployment history, training completions — was stored in physical paper folders inside filing cabinets.

Retrieving a single file took between 5 to 15 minutes. Records were misfiled, damaged, or lost. Sharing information between branches required physically moving folders or making photocopies. There was no access control — anyone with physical access to the cabinet could read or modify sensitive employee data.

As the organisation's workforce grew, these inefficiencies compounded. The HR department was spending the majority of its time on administrative search tasks instead of strategic workforce planning.

02 — The Solution

A secure, centralized HRMS built for security operations.

I designed and am developing a custom Human Resource Management System tailored specifically to GardaWorld's operational structure — not a generic off-the-shelf tool, but a platform built around how a security company actually works.

The system is built on a three-tier architecture: a React frontend, a Laravel REST API backend, a MySQL relational database, and AWS S3 for secure document storage. Role-based access control ensures that HR officers, supervisors, and administrators each see only what they are permitted to access.

  • Employee profile management with full document upload and retrieval
  • Recruitment and applicant tracking module with status notifications
  • Attendance and leave management with supervisor approval workflows
  • Payroll processing module with automated net salary calculation
  • Training and performance evaluation tracking
  • Audit logs recording every record access and modification
  • Multi-branch support with centralized data access
03 — Security Considerations

Built with cybersecurity at the foundation.

Employee records in a security firm are extremely sensitive — national IDs, clearance letters, disciplinary files, and salary information. My background in networking and cybersecurity directly informed how this system was designed.

Every API endpoint is authenticated via Laravel Sanctum. Documents are stored in private AWS S3 buckets with signed URLs for time-limited access. Input validation guards against SQL injection and XSS. Database access is role-scoped so no single user has unrestricted access to the full dataset.
